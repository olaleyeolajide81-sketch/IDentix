# Contributing to IDentix

Thank you for your interest in contributing to IDentix! This document provides guidelines and information for contributors.

## How to Contribute

### 1. Setting Up Your Development Environment

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bugfix
4. Set up your development environment

### 2. Development Workflow

- Follow the existing code style and patterns
- Write tests for new functionality
- Ensure all tests pass before submitting
- Update documentation as needed

### 3. Submitting Changes

1. Push your changes to your fork
2. Create a pull request with a clear description
3. Link relevant issues in your PR
4. Wait for code review

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and grow
- Follow the project's technical standards

## Issue Categories

### 🐛 Bug Reports
- Clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Environment details

### ✨ Feature Requests
- Use case description
- Proposed implementation
- Potential impact

### 📚 Documentation
- Missing or unclear documentation
- Examples and tutorials
- API reference improvements

### 🔧 Development
- Code improvements
- Performance optimizations
- Testing enhancements

## Development Guidelines

### Smart Contracts (Stellar/Soroban)
- Use Rust best practices
- Include comprehensive tests
- Document contract interfaces
- Consider gas optimization

### Frontend (React/TypeScript)
- Follow TypeScript best practices
- Use component-based architecture
- Include proper error handling
- Ensure responsive design

### Backend (Node.js/TypeScript)
- Follow REST API principles
- Include input validation
- Use proper error handling
- Include comprehensive logging

## Testing

- Unit tests for all components
- Integration tests for APIs
- End-to-end tests for critical flows
- Smart contract testing with Stellar Testnet

## Release Process

1. Update version numbers
2. Update changelog
3. Create release notes
4. Tag and publish release

## Getting Help

- Check existing issues and documentation
- Join our community discussions
- Ask questions in appropriate channels
- Participate in code reviews

Thank you for contributing to IDentix! 🚀
